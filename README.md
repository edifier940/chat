# SupportXMR Monitor

Веб-приложение для мониторинга воркеров на пуле [SupportXMR](https://www.supportxmr.com/).
Показывает общий хешрейт и график, список воркеров со статусами, отслеживает
**новые** и **пропавшие** воркеры (которые не появлялись дольше заданного порога,
например > 1 часа), а также даёт фильтр, поиск и сортировку.

Адрес по умолчанию:

(меняется в настройках без перезапуска).

## Возможности

- **Дашборд**: общий хешрейт, кол-во воркеров (онлайн / новые / пропали),
  принятые/отклонённые шары, сумма к выплате и выплачено.
- **График хешрейта пула** за ~10 часов (area-chart, тултипы).
- **Список воркеров**: текущий хешрейт, доля в общем хешрейте, всего хешей,
  время последней активности, цветовой статус (онлайн / пропал) и бейдж `NEW`.
- **Новые воркеры**: первое появление запоминается локально (localStorage),
  воркер считается новым в течение настраиваемого окна.
- **Пропавшие воркеры**: если `lts` (last-seen) старше порога (по умолчанию 1 час) —
  воркер помечается как «пропал».
- **Фильтр** (Все / Онлайн / Новые / Пропали), **поиск** по имени и
  **сортировка** (хешрейт, имя, последняя активность, всего хешей, дата появления)
  с переключением направления.
- **Детальное окно** по клику на воркера: текущий / средний / пиковый хешрейт и
  персональный график.
- Автообновление с настраиваемым интервалом (10 сек … 5 мин) и ручное обновление.

## Запуск

```bash
npm install
npm run dev      # дев-сервер на http://localhost:5173
```

Продакшн-сборка:

```bash
npm run build    # статика в dist/
npm run preview  # локальный предпросмотр сборки
```

Бэкенд не нужен — приложение ходит в публичный API SupportXMR напрямую
(CORS на стороне пула открыт).

## Используемые эндпоинты API

| Назначение | Эндпоинт |
| --- | --- |
| Общая статистика | `/api/miner/{addr}/stats` |
| Имена всех воркеров | `/api/miner/{addr}/identifiers` |
| Снимок всех воркеров | `/api/miner/{addr}/stats/allWorkers` |
| Статистика воркера | `/api/miner/{addr}/stats/{worker}` |
| График хешрейта пула | `/api/miner/{addr}/chart/hashrate` |
| График хешрейта воркера | `/api/miner/{addr}/chart/hashrate/{worker}` |

> Замечание: `stats/allWorkers` у пула работает нестабильно и периодически
> возвращает только запись `global`. Поэтому полный список имён всегда берётся из
> `identifiers`, а недостающие воркеры дозапрашиваются поштучно через
> `stats/{worker}` (с ограничением на число одновременных запросов).
